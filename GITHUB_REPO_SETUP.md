## 🏷️ Repository Settings

### General Settings

1. **Template repository**: ❌ Unchecked (unless you want others to use it as template)
2. **Require contributors to sign off on web-based commits**: ✅ Recommended
3. **Sponsorship**: ❌ Unchecked (unless you want GitHub Sponsors)

### Features to Enable

- ✅ **Wikis** - For extended documentation
- ✅ **Issues** - For bug tracking and feature requests  
- ✅ **Discussions** - For community Q&A
- ✅ **Projects** - For project management
- ❌ **Preserve this repository** - Not needed for active projects

### Pull Requests

- ✅ **Allow merge commits**
- ✅ **Allow squash merging**
- ✅ **Allow rebase merging**
- ✅ **Always suggest updating pull request branches**
- ✅ **Automatically delete head branches**

## 🎨 Repository Social Preview

Upload a custom social preview image (1280x640px recommended):

1. Go to **Settings** → **Options**
2. Scroll to **Social preview**
3. Upload an image showing:
   - Project logo
   - Project name "SmartSched"
   - Tagline "AI-Powered Timetable Generation"
   - Tech stack icons (React, Node, MongoDB)

## 📊 GitHub Pages (Optional)

You can enable GitHub Pages to host documentation:

1. Go to **Settings** → **Pages**
2. Source: **Deploy from a branch**
3. Branch: **main** / **docs** folder
4. Theme: Choose a Jekyll theme
